APIs on the Client

APIs on the Client

  • Just regular HTTP calls from JavaScript
  • To be clear - NOT SPECIFIC TO JAMSTACK
  • A simple way to add in awesomeness!
  • "Just works" (mostly...)

APIs on the Client - Cons

  • CORS
  • Anonymous access
  • Can't modify data (kinda)

API Proxies

API Proxies

  • Fancy way of saying a midpoint between your client-side code and the API
  • Serverless is a great use case
  • You can hide keys
  • You can shape the result
  • You can switch providers
  • You can add caching

Demo

Demo2 uses a serverless proxy to SW api, it shapes the response show the difference in size show netlify version in demo3

API Proxies - Cons

  • You have to write code (maybe)
  • You may need yet another service

Using the Build Process

  • A URL trigger/A GitHub commit/etc
  • Well suited for dynamic data not changing "often"
  • Options depend on platform and static site generator
"often" will vary site to site. Product availability may update every day.
